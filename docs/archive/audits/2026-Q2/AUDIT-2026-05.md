## 1. Stack i arquitectura

- **Gràfics:** `recharts` `^3.8.1` (chunk vendor separat a `vite.config.js`).

La infraestructura Stripe (Checkout + webhook) està implementada des de Fase 4.5 (`api/main.py` línies 232–632: `/checkout`, `/webhooks/stripe`, vars `STRIPE_PRICE_ID`, `STRIPE_WEBHOOK_SECRET`, `STRIPE_SECRET_KEY`). Disparadors: push i PR a `main`. 3 jobs paral·lels:
1. **Build & lint:** `npm install` → `npm run build` → "bundle sanity" (grep hexa colors `0047ba`, `cf3339`, `f1c22f`, `427c42`, `ffffff`, `111111` a `dist/assets/*.js` per detectar trencaments de tokens mm-design).

3. **Backend tests:** Python 3.11 → `pip install -r api/requirements.txt pytest pytest-asyncio httpx` → `cd api && pytest tests/ -v`.

**No documentat com a fet / no trobat al repo.** Apareix només com a **fase pendent** a `ROADMAP.md` línies 105–109 amb la justificació: GitHub Pages no permet `cache-control` llarg ni Brotli ni HTTP/3, i Cloudflare donaria aquests beneficis més edge CDN i tier gratuït. Estimació documentada: 2–4h. **No** hi ha cap workflow, branca, configuració `wrangler.toml`, `_headers`, `_redirects` ni commit que indiqui que la migració s'hagi començat.
